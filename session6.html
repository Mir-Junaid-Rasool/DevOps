1Ô∏è‚É£ GIT Merge

What is Git Merge?

Git Merge combines changes from different branches. It creates a new commit that brings together the histories of both branches.

# Basic merge syntax git merge <branch-name> # Example: Merge feature branch into main git checkout main git merge feature-branch

Complete Merge Workflow

# 1. Create and switch to new branch git checkout -b feature-login # 2. Make changes and commit echo "Login feature" > login.html git add login.html git commit -m "Add login feature" # 3. Switch back to main git checkout main # 4. Merge feature into main git merge feature-login Updating a1b2c3d..e5f6g7h Fast-forward login.html | 1 + 1 file changed, 1 insertion(+)

Types of Merges

Type Description When It Happens
Fast-Forward Simply moves pointer forward When no diverging changes exist
Three-Way Merge Creates new merge commit When both branches have changes
Squash Merge Combines all commits into one When you want clean history

Handling Merge Conflicts

‚ö†Ô∏è What is a Merge Conflict?

A conflict occurs when Git can't automatically merge changes because the same lines were modified in both branches.

Resolving Merge Conflicts

# Attempt merge git merge feature-branch Auto-merging index.html CONFLICT (content): Merge conflict in index.html Automatic merge failed; fix conflicts and then commit the result. # Check conflict status git status Unmerged paths: both modified: index.html # Open index.html - you'll see: <<<<<<< HEAD <h1>Welcome to Main Branch</h1> ======= <h1>Welcome to Feature Branch</h1> >>>>>>> feature-branch # Edit file to keep what you want # Remove conflict markers <h1>Welcome to Our Website</h1> # Mark as resolved git add index.html git commit -m "Resolve merge conflict"

2Ô∏è‚É£ GIT Rebase

What is Git Rebase?

Git Rebase rewrites commit history by moving commits to a new base. It creates a linear, cleaner history.

# Basic rebase syntax git rebase <base-branch> # Example: Rebase feature onto main git checkout feature-branch git rebase main

Merge vs Rebase

Aspect Merge Rebase
History Preserves complete history Creates linear history
Commits Creates merge commit Replays commits on top
Use Case Integrating feature branches Keeping feature branch updated
Safety ‚úÖ Safe for shared branches ‚ö†Ô∏è Never rebase public branches
Conflicts Resolve once May resolve multiple times

‚ö†Ô∏è Golden Rule of Rebase

Never rebase commits that have been pushed to a public/shared repository!

Only rebase local commits that haven't been shared with others.

Interactive Rebase

# Interactive rebase last 3 commits git rebase -i HEAD~3 # Editor opens with: pick a1b2c3d Add login form pick e5f6g7h Fix validation pick i9j0k1l Update styles # Commands available: # pick (p) = use commit # reword (r) = use commit, but edit message # edit (e) = use commit, but stop for amending # squash (s) = combine with previous commit # drop (d) = remove commit # Example: Squash last two commits pick a1b2c3d Add login form squash e5f6g7h Fix validation squash i9j0k1l Update styles

3Ô∏è‚É£ GIT Stash

What is Git Stash?

Git Stash temporarily saves your uncommitted changes, allowing you to switch contexts without committing incomplete work.

üí° When to Use Stash?

  • Need to switch branches but work isn't ready to commit
  • Want to pull latest changes without committing
  • Emergency bug fix needed on another branch
  • Experiment with code without losing current work

Basic Stash Commands

# Save changes to stash git stash Saved working directory and index state WIP on main # Stash with message git stash save "Work in progress on login" # List all stashes git stash list stash@{0}: WIP on main: a1b2c3d Add login stash@{1}: WIP on main: e5f6g7h Fix bug # Apply most recent stash git stash apply # Apply specific stash git stash apply stash@{1} # Apply and remove from stash list git stash pop # Delete stash git stash drop stash@{0} # Clear all stashes git stash clear # Show stash contents git stash show

Practical Stash Workflow

# Working on feature branch git checkout feature-payment # Making changes... # Urgent bug reported! Need to switch to main git stash save "Payment integration incomplete" # Switch to main and fix bug git checkout main # Fix bug, commit # Return to feature work git checkout feature-payment git stash pop On branch feature-payment Changes not staged for commit: modified: payment.js Dropped refs/stash@{0} # Continue working!

4Ô∏è‚É£ GIT Reset

What is Git Reset?

Git Reset moves the current branch pointer to a different commit, effectively "undoing" commits.

Three Types of Reset

Type Working Directory Staging Area Commits
--soft Unchanged Unchanged Reset
--mixed (default) Unchanged Reset Reset
--hard Reset Reset Reset
# Soft reset - keep changes staged git reset --soft HEAD~1 # Mixed reset - keep changes unstaged (default) git reset HEAD~1 git reset --mixed HEAD~1 # Hard reset - discard all changes ‚ö†Ô∏è DANGEROUS! git reset --hard HEAD~1 # Reset to specific commit git reset --hard a1b2c3d # Unstage file (mixed reset) git reset HEAD file.txt

‚ö†Ô∏è Warning: Git Reset is Destructive

git reset --hard permanently deletes changes. Use with extreme caution!

Never reset commits that have been pushed to remote repositories.

Common Reset Scenarios

# Scenario 1: Undo last commit, keep changes git reset --soft HEAD~1 # Changes still staged, can re-commit # Scenario 2: Unstage files git add . git reset # Files unstaged but changes preserved # Scenario 3: Discard all local changes git reset --hard HEAD # ‚ö†Ô∏è All uncommitted changes DELETED! # Scenario 4: Go back 3 commits git reset --hard HEAD~3 # Last 3 commits removed, changes deleted

5Ô∏è‚É£ GIT Checkout

What is Git Checkout?

Git Checkout switches between branches or restores files. It's a versatile command with multiple uses.

Checkout Uses

  • Switch between branches
  • Create and switch to new branch
  • Restore files from specific commits
  • Discard local changes in files
  • View old versions of files

Common Checkout Commands

# Switch to existing branch git checkout branch-name # Create and switch to new branch git checkout -b new-branch # Discard changes in file git checkout -- file.txt # Restore file from specific commit git checkout a1b2c3d -- file.txt # Checkout specific commit (detached HEAD) git checkout a1b2c3d # Return to branch from detached HEAD git checkout main # Checkout remote branch git checkout -b local-branch origin/remote-branch

üí° Git Switch & Restore (Modern Alternatives)

Git 2.23+ introduced clearer commands:

# Switch branches (replaces checkout for branches) git switch branch-name git switch -c new-branch # Restore files (replaces checkout for files) git restore file.txt git restore --source=HEAD~2 file.txt

Checkout Scenarios

# Scenario 1: Accidentally modified file git checkout -- config.js Updated 1 path from the index # Scenario 2: Need code from old commit git checkout abc123 -- old-feature.js # Scenario 3: Explore old version git checkout HEAD~5 Note: switching to 'HEAD~5'. You are in 'detached HEAD' state... git checkout main # Return to main

6Ô∏è‚É£ GIT Fetch & Pull Revisited

Git Fetch

Downloads changes from remote but doesn't merge them.

# Fetch all branches git fetch origin # Fetch specific branch git fetch origin main # View fetched changes git log origin/main # Merge fetched changes git merge origin/main

Git Pull (Fetch + Merge)

# Pull = Fetch + Merge git pull origin main # Pull with rebase instead of merge git pull --rebase origin main
Command What It Does When to Use
git fetch Download only Want to review before merging
git pull Download + merge Quick sync with remote
git pull --rebase Download + rebase Want linear history
