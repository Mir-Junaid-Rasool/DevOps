Part 1: Jenkins - CI/CD Automation (90 minutes)

üîß

What is Jenkins?

Jenkins is an open-source automation server that enables Continuous Integration and Continuous Delivery. It automates the building, testing, and deployment of applications.

Key Features:

  • ‚úÖ Free and open source
  • ‚úÖ 1800+ plugins for integration with almost any tool
  • ‚úÖ Distributed builds (master-slave architecture)
  • ‚úÖ Easy installation and configuration
  • ‚úÖ Supports all major version control systems
  • ‚úÖ Pipeline as Code (Jenkinsfile)

Jenkins Installation (Windows)

Prerequisites:

  • Java JDK 11 or later installed
  • At least 256 MB RAM (1 GB recommended)
  • 10 GB of disk space

Installation Steps:

  1. Download Jenkins:
    • Visit https://www.jenkins.io/download/
    • Download jenkins.war file or Windows installer
  2. Install Java (if not installed):
    # Check Java version java -version # If not installed, download from Oracle or use OpenJDK
  3. Run Jenkins WAR file:
    # Navigate to download folder cd Downloads # Run Jenkins java -jar jenkins.war # Or specify port (default is 8080) java -jar jenkins.war --httpPort=9090
  4. Access Jenkins:
    • Open browser: http://localhost:8080
    • You'll see "Unlock Jenkins" screen
  5. Get Initial Admin Password:
    # Password location (Windows): C:\Users\YourUsername\.jenkins\secrets\initialAdminPassword # Or check console output for password
  6. Install Suggested Plugins:
    • Click "Install suggested plugins"
    • Wait for plugins to install (~5 minutes)
  7. Create Admin User:
    • Enter username, password, email
    • Click "Save and Continue"
  8. Jenkins is Ready!
    • You'll see the Jenkins Dashboard

Jenkins Installation (Linux/Ubuntu)

# Update package index sudo apt update # Install Java sudo apt install openjdk-11-jdk -y # Add Jenkins repository curl -fsSL https://pkg.jenkins.io/debian-stable/jenkins.io.key | sudo tee \ /usr/share/keyrings/jenkins-keyring.asc > /dev/null echo deb [signed-by=/usr/share/keyrings/jenkins-keyring.asc] \ https://pkg.jenkins.io/debian-stable binary/ | sudo tee \ /etc/apt/sources.list.d/jenkins.list > /dev/null # Install Jenkins sudo apt update sudo apt install jenkins -y # Start Jenkins service sudo systemctl start jenkins sudo systemctl enable jenkins # Check status sudo systemctl status jenkins # Get initial password sudo cat /var/lib/jenkins/secrets/initialAdminPassword

Common Uses of Jenkins

Use Case 1: Automated Testing

Scenario: Every time a developer pushes code to GitHub, automatically run all tests.

Setup:

  1. Create a new Jenkins job
  2. Connect to GitHub repository
  3. Configure webhook (GitHub notifies Jenkins on push)
  4. Add build step: mvn test (for Java) or npm test (for Node.js)
  5. If tests fail, notify team via email/Slack

Result: No broken code reaches production!

Use Case 2: Continuous Deployment

Scenario: Deploy application to server automatically after successful tests.

Jenkins Pipeline:

pipeline { agent any stages { stage('Build') { steps { sh 'mvn clean package' } } stage('Test') { steps { sh 'mvn test' } } stage('Deploy') { steps { sh 'scp target/app.war user@server:/opt/tomcat/webapps/' } } } }

Use Case 3: Scheduled Jobs

Scenario: Run database backup every night at 2 AM.

Configuration:

  • Build Triggers ‚Üí Build periodically
  • Cron: 0 2 * * * (At 2:00 AM every day)
  • Execute shell: pg_dump mydb > backup.sql

Part 2: Docker - Containerization (90 minutes)

üê≥

What is Docker?

Docker is a containerization platform that packages applications and their dependencies into lightweight, portable containers that run consistently across different environments.

Key Features:

  • ‚úÖ Lightweight (shares OS kernel, unlike VMs)
  • ‚úÖ Fast startup (seconds vs minutes for VMs)
  • ‚úÖ Portable (runs anywhere: laptop, cloud, on-premise)
  • ‚úÖ Isolated environments for each application
  • ‚úÖ Version control for infrastructure
  • ‚úÖ Efficient resource usage

Docker Installation (Windows)

Prerequisites:

  • Windows 10/11 64-bit: Pro, Enterprise, or Education
  • Hardware virtualization enabled in BIOS
  • WSL 2 (Windows Subsystem for Linux)

Installation Steps:

  1. Enable WSL 2:
    # Open PowerShell as Administrator wsl --install # Restart computer
  2. Download Docker Desktop:
    • Visit https://www.docker.com/products/docker-desktop
    • Download Docker Desktop for Windows
    • Run the installer
  3. Install and Restart:
    • Follow installation wizard
    • Restart computer when prompted
  4. Verify Installation:
    # Open Command Prompt or PowerShell docker --version # Run test container docker run hello-world # You should see "Hello from Docker!" message

Docker Installation (Linux/Ubuntu)

# Update package index sudo apt update # Install dependencies sudo apt install apt-transport-https ca-certificates curl software-properties-common -y # Add Docker's official GPG key curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg # Add Docker repository echo "deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null # Install Docker sudo apt update sudo apt install docker-ce docker-ce-cli containerd.io -y # Start Docker service sudo systemctl start docker sudo systemctl enable docker # Verify installation docker --version sudo docker run hello-world # Add your user to docker group (to run without sudo) sudo usermod -aG docker $USER # Log out and log back in for this to take effect

Essential Vagrant Commands

# Initialize Vagrant project vagrant init ubuntu/focal64 # Start the VM vagrant up # SSH into the VM vagrant ssh # Check VM status vagrant status # Suspend the VM vagrant suspend # Resume the VM vagrant resume # Restart the VM vagrant reload # Stop the VM vagrant halt # Destroy the VM vagrant destroy

Common Uses of Vagrant

Use Case: Team Development Environment

Problem: "It works on my machine" syndrome

Solution with Vagrant:

# Create Vagrantfile Vagrant.configure("2") do |config| config.vm.box = "ubuntu/focal64" config.vm.network "forwarded_port", guest: 80, host: 8080 # Install software automatically config.vm.provision "shell", inline: <<-SHELL apt-get update apt-get install -y nginx nodejs npm SHELL end # Every team member runs: vagrant up # Everyone gets identical environment!

Part 4: Configuration Management Tools (60 minutes)

What is Configuration Management?

Configuration management tools automate the setup and maintenance of servers. Instead of manually installing software on each server, you write code that does it automatically.

üî¥

Ansible

Type: Agentless (no software needed on managed servers)

Language: YAML

Best For: Simple automation, cloud provisioning

Installation (Linux):

# Ubuntu/Debian sudo apt update sudo apt install ansible -y # Verify ansible --version

Example Playbook:

# install_nginx.yml --- - hosts: webservers become: yes tasks: - name: Install Nginx apt: name: nginx state: present - name: Start Nginx service: name: nginx state: started # Run playbook ansible-playbook install_nginx.yml

Pros:

  • ‚úÖ Easy to learn (YAML syntax)
  • ‚úÖ No agent required
  • ‚úÖ Uses SSH (secure)
  • ‚úÖ Fast to get started

Use Case: Deploy to 100 Servers

Instead of manually SSH-ing to 100 servers to install updates, run one Ansible command!

üü°

Puppet

Type: Agent-based (requires Puppet agent on servers)

Language: Puppet DSL (Domain Specific Language)

Best For: Large enterprises, complex infrastructures

Installation:

# Install Puppet Master (Ubuntu) wget https://apt.puppet.com/puppet7-release-focal.deb sudo dpkg -i puppet7-release-focal.deb sudo apt update sudo apt install puppetserver -y # Install Puppet Agent sudo apt install puppet-agent -y # Start Puppet Server sudo systemctl start puppetserver sudo systemctl enable puppetserver

Example Manifest:

# site.pp node 'webserver' { package { 'nginx': ensure => installed, } service { 'nginx': ensure => running, enable => true, } }

Pros:

  • ‚úÖ Mature and stable
  • ‚úÖ Strong enterprise support
  • ‚úÖ Excellent reporting
  • ‚úÖ Large module library
üü†

Chef

Type: Agent-based

Language: Ruby DSL

Best For: Complex workflows, cloud automation

Installation:

# Install Chef Workstation (Ubuntu) wget https://packages.chef.io/files/stable/chef-workstation/21.10.640/ubuntu/20.04/chef-workstation_21.10.640-1_amd64.deb sudo dpkg -i chef-workstation_*.deb # Verify chef --version

Example Recipe:

# nginx_recipe.rb package 'nginx' do action :install end service 'nginx' do action [:enable, :start] end file '/var/www/html/index.html' do content '<h1>Hello from Chef!</h1>' end

Pros:

  • ‚úÖ Powerful and flexible
  • ‚úÖ Strong testing framework
  • ‚úÖ Good for compliance automation
  • ‚úÖ Cloud-native features

Practical Comparison: Same Task, Different Tools

üéØ Task: Install Nginx on 10 Servers

Manual Way (Traditional):

  1. SSH to server 1: ssh user@server1
  2. Run: sudo apt install nginx
  3. Start service: sudo systemctl start nginx
  4. Repeat for server 2, 3, 4... 10

Time: ~30 minutes

Risk: Human error, inconsistency

With Ansible:

# inventory.ini [webservers] server1.example.com server2.example.com server3.example.com server4.example.com server5.example.com server6.example.com server7.example.com server8.example.com server9.example.com server10.example.com # playbook.yml --- - hosts: webservers become: yes tasks: - name: Install Nginx apt: name: nginx state: present update_cache: yes - name: Start Nginx service: name: nginx state: started enabled: yes # Run on all 10 servers at once! ansible-playbook -i inventory.ini playbook.yml # Time: 2 minutes # Result: All servers configured identically

Tool Selection Guide

When to Use What?

Scenario Recommended Tool Why?
Build and test code automatically Jenkins Purpose-built for CI/CD, extensive plugins
Package application with dependencies Docker Lightweight, portable, consistent environments
Create identical dev environments for team Vagrant Full VMs, works with any OS
Quick server configuration (cloud) Ansible Simple, agentless, fast to learn
Large enterprise with compliance needs Puppet Mature, excellent reporting, enterprise support
Complex infrastructure as code Chef Powerful, flexible, strong testing

üí° Pro Tip: Use Tools Together!

Modern DevOps workflows combine multiple tools:

  • Development: Vagrant (local VMs) + Docker (containers)
  • CI/CD: Jenkins (automation) + Docker (build environments)
  • Deployment: Ansible (server config) + Docker (app deployment)
  • Infrastructure: Terraform (provision) + Ansible (configure)

üéØ Hands-On Lab Activity (20 minutes)

Lab 1: Jenkins - Create Your First Job

  1. Install Jenkins (if not already done)
  2. Access Jenkins at http://localhost:8080
  3. Click "New Item" ‚Üí Enter name "Hello-World" ‚Üí Select "Freestyle project"

  2. Run: docker run -d -p 8080:80 nginx
  3. Open browser: http://localhost:8080
  4. You should see "Welcome to nginx!" page
  5. Stop container: docker ps (get container ID)
  6. docker stop [container-id]

Lab 3: Ansible - Configure a Local Server

  1. Install Ansible (Linux/WSL): sudo apt install ansible -y
  2. Create file playbook.yml with Nginx installation tasks
  3. Create inventory.ini with localhost
  4. Run: ansible-playbook -i inventory.ini playbook.yml
  5. Verify Nginx is installed: systemctl status nginx

Common Uses of Docker

Use Case 1: Running a Web Server

Scenario: Run Nginx web server in seconds

# Pull and run Nginx docker run -d -p 8080:80 --name webserver nginx # Access at http://localhost:8080 # Stop and remove docker stop webserver docker rm webserver

Why Docker? No need to install Nginx on your system, no configuration conflicts!

Use Case 2: Development Environment

Scenario: Create isolated database for testing

# Run MySQL container docker run -d \ --name mysql-dev \ -e MYSQL_ROOT_PASSWORD=password123 \ -e MYSQL_DATABASE=testdb \ -p 3306:3306 \ mysql:8.0 # Connect to MySQL docker exec -it mysql-dev mysql -uroot -p # When done, remove everything docker stop mysql-dev docker rm mysql-dev

Benefit: Clean database for each test, no leftover data!
